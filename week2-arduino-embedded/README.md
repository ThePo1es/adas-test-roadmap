# Week 2 — 아두이노 임베디드 C + 센서/액추에이터

**목표:** 진짜 MCU 위에서 C를 쓴다. 화면 출력이 아니라 모터가 돌고 부저가 울린다. 제어기(ECU)의 축소판을 손으로 만든다.
**왜 아두이노 C인가:** 공고의 C/C++ 요구 + 제어기 개발/검증은 결국 임베디드 C. 추상적 C보다 보드 위 C가 훨씬 잘 붙는다.
**준비물:** 아두이노, HC-SR04, DC모터+드라이버(L9110/L298N), 부저, LED, 가변저항

---

## Day 1 (월) — 아두이노 C 기초 + 디지털 출력

| 시간 | 할 일 |
|------|------|
| 0:00~1:00 | 아두이노 IDE 설치, `setup()`/`loop()` 구조, `pinMode`/`digitalWrite`. 컴파일→업로드 흐름 |
| 1:00~2:00 | C 문법 vs 파이썬 차이 정리(정적타입, 세미콜론, 중괄호). LED 점멸, `millis()`로 delay 없이 점멸(블로킹 이해) |
| 2:00~4:30 | 가변저항(아날로그 입력) `analogRead` → 값에 따라 LED 밝기 `analogWrite`(PWM). 시리얼 모니터로 값 확인 |
| 4:30~5:00 | 커밋 |

**완료 기준:** PWM으로 LED 밝기를 손잡이(가변저항)로 조절.
