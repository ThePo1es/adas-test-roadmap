# 자율주행/차량 시험 엔지니어 지망 — 실물 보드 4주 부트캠프

> 목표 직무: 현대차 자율주행/주차 시스템 **시험·검증** (실차 검증, Test Case 개발, 기능안전/사이버보안)
> 소속 동아리: 차량보안 + 차량 임베디드
> 대상: 자동차IT공학 2학년 / 코딩은 LLM 없이는 기초 수준 / **손으로 만지는 걸 좋아함**
> 기간: 4주 × 주 7일 × 5시간 (6일 실습 + 1일 복습/정리)
> 보유 장비: 라즈베리파이, 아두이노 (+ 추가 부품은 `HARDWARE.md` 참고, 3~4만원)

## 핵심 컨셉

책상에 **실제 3노드 CAN 버스**를 만든다. 아두이노 2대가 차량 ECU(차속 센서 / AEB 제어기),
라즈베리파이가 시험·분석·공격 장비. 이 위에서 임베디드 개발 → CAN 통신 → 보안 시험 → 자동긴급제동(AEB)
시험까지 **전부 물리적으로** 한 바퀴 돈다.

4주 뒤 결과물: **초음파로 전방 거리를 재서 실제 모터를 멈추는 AEB 테스트베드 + CAN 스니핑/스푸핑 보안 시험 + 자동 시험 리포트.**

## 왜 이 순서인가

시험 직무의 실제 사이클은 ①요구사양 → ②Test Case → ③시험 실행 → ④데이터 분석 → ⑤리포트다.
여기에 동아리(임베디드+보안)를 얹어, 코드를 화면이 아니라 **보드 위에서** 검증하며 배운다.

| 주차 | 폴더 | 주제 | 손에 잡히는 결과물 |
|---|---|---|---|
| 1 | `week1-pi-python/` | 라즈베리파이 + Python + Git + GPIO | LED/버튼/초음파를 파이썬으로 제어 |
| 2 | `week2-arduino-embedded/` | 아두이노 임베디드 C + 센서/액추에이터 | 거리센서→모터/부저, 시리얼 로깅 |
| 3 | `week3-can-bus-security/` | 실물 CAN 버스 구축 + 스니핑/스푸핑 + 안전규격 | 3노드 CAN 통신, Pi로 프레임 주입 |
| 4 | `week4-capstone-aeb-testbed/` | 캡스톤: 물리 AEB 테스트베드 + 시험 자동화 + 보안 시험 | 요구사양→TC→실차(모형)시험→리포트 |

## 절대 규칙

1. **LLM 코드 생성 금지.** 검색·데이터시트·공식 문서는 허용. LLM은 "개념 질문"과 "내 코드 리뷰"에만.
2. 매일 커밋 + **작동 영상/사진 1장**을 커밋에 포함(포트폴리오이자 시험 증적 습관).
3. 배선은 항상 **전원 끄고**. 연결 후 사진 찍고 → 재확인 → 전원.
4. 막히면 30분 버티고 기록 남기고 넘어간다. 7일차에 주간 회고(`TIL.md`).

## 하루 5시간 배분

- 1h 개념/데이터시트 학습
- 3h 보드 실습 (배선 + 코딩, LLM 금지)
- 0.5h 디버깅/측정
- 0.5h 커밋 + 사진/메모

## 4주 이후

- **프로그래머스 Lv.2 도달까지 매일 1~2문제** (알고리즘 근육. 백준 부활 시 병행)
- STM32(Nucleo)로 넘어가 HAL/레지스터 레벨 CAN — 양산 ECU에 더 가까움
- 동아리 프로젝트: UDS(ISO 14229) 진단, SecOC 메시지 인증 구현, 실차 OBD-II 스캔
- 자율주행/임베디드/보안 경진대회 참가 (공고 "대회 참여/입상" 우대)
- MATLAB/Simulink Onramp (공고의 dSPACE/Simulink 요구 대응)
