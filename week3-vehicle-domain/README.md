## Day 5 (금) — 기능안전(ISO 26262) / 운용안전(ISO 21448) / 사이버보안(ISO 21434) → 과제 4

| 시간 | 할 일 |
|------|------|
| 0:00~1:30 | ISO 26262: V모델 개발 프로세스, 좌측(요구→설계) 우측(시험→검증)의 대응 관계. **시험 직무는 V모델 오른쪽 전체** |
| 1:30~2:30 | ASIL 등급: S(심각도)×E(노출)×C(제어가능성)로 A~D 결정. 예제로 AEB 오작동의 ASIL 직접 판정해 보기 |
| 2:30~3:15 | ISO 21448(SOTIF): 26262와의 차이 — "고장(fault)"이 아니라 "성능 한계·오사용"을 다룸. 센서가 정상인데 안개로 못 보는 경우가 SOTIF |
| 3:15~4:00 | ISO/SAE 21434(차량 사이버보안, 공고에 명시): TARA(위협분석·리스크평가) 개념, 26262와의 관계(안전 vs 보안). **동아리 활동과 직결되는 규격** |
| 4:00~4:30 | **과제 4: `assignments/safety_summary.md`** — 세 규격 각 10줄 요약 + "AEB에 적용하면 필요한 시험(보안 포함) 3가지"씩 |
| 4:30~5:00 | 커밋 |

**완료 기준:** "26262 vs 21448 vs 21434"를 각각 예시 1개씩 들어 설명할 수 있다 (고장/성능한계/공격).
