# 하드웨어 준비 (Bill of Materials)

이 로드맵은 **실물 보드로 CAN 버스를 직접 만들고, 그 위에서 시험·보안 실습**을 한다.
이미 보유: 라즈베리파이, 아두이노(Uno/Nano 계열). 아래는 추가로 살 것들.

## 꼭 필요한 것 (핵심 실습용)

| 부품 | 수량 | 용도 | 대략가(원) |
|------|------|------|-----------|
| MCP2515 CAN 모듈 (TJA1050 트랜시버 포함) | **3** | CAN 노드용. 아두이노×2 + 라즈베리파이×1 | 3,000~5,000/개 |
| 브레드보드 (830핀) | 2 | 회로 구성 | 3,000 |
| 점퍼 와이어 (M-M, M-F 각 40p) | 1세트 | 배선 | 4,000 |
| HC-SR04 초음파 거리센서 | 1 | **AEB 캡스톤의 "전방 장애물 거리" 입력** | 1,500 |
| DC 기어모터 + 바퀴 + L9110/L298N 드라이버 | 1세트 | AEB "차량 구동/제동" 액추에이터 | 5,000 |
| 능동 부저(피에조) | 1 | AEB 경고음 | 500 |
| LED(빨/노/초) + 220Ω 저항 | 각 3 | 상태 표시 | 1,000 |
| 푸시버튼 + 가변저항(10kΩ) | 각 2 | 입력(스로틀 시뮬레이션) | 1,000 |
| 120Ω 저항 | 2 | CAN 버스 양단 종단저항 (모듈에 점퍼로 내장된 경우 많음) | 200 |

**합계 대략 3~4만원.** 동아리에 MCP2515나 센서가 이미 있으면 그만큼 뺀다.

## 있으면 좋은 것 (선택)

- **라즈베리파이용 CAN HAT** (MCP2515 기반): 배선 없이 Pi에 바로 장착. 위 모듈 대신 사용 가능(권장) — 8,000~15,000원
- **8채널 로직 애널라이저** (FX2 클론): SPI/CAN 파형 디버깅용. 배선 문제 잡을 때 신세계 — 8,000원
- 점퍼 정리용 소켓, 미니 프레임/RC카 섀시(모터 장착용)

## 실습 아키텍처 (4주 뒤 완성형)

```
        CAN_H ──┬──────────────┬──────────────┬──
        CAN_L ──┼──────────────┼──────────────┼──
              [120Ω]         (노드)         [120Ω]
                │              │              │
          ┌─────┴─────┐  ┌─────┴─────┐  ┌─────┴──────┐
          │ Arduino#1 │  │ Arduino#2 │  │ Raspberry  │
          │ "차속 ECU"│  │ "AEB ECU" │  │ Pi: 분석/  │
          │ HC-SR04   │  │ 모터+부저 │  │ 공격/시험  │
          │ →거리송신 │  │ →제동수행 │  │ can-utils  │
          └───────────┘  └───────────┘  └────────────┘
```

- Arduino#1: 초음파로 전방 거리 측정 → CAN으로 방송
- Arduino#2: 거리 프레임 수신 → TTC 계산 → 임계 이하면 모터 정지 + 부저 (AEB)
- 라즈베리파이: 리눅스 can-utils로 버스 스니핑/디코딩, 시험 자동화, **가짜 거리 프레임 주입(보안 시험)**

## 배선 참고 (MCP2515 ↔ 아두이노, SPI)

```
MCP2515   Arduino Uno
  VCC  →  5V
  GND  →  GND
  CS   →  D10
  SO   →  D12 (MISO)
  SI   →  D11 (MOSI)
  SCK  →  D13 (SCK)
  INT  →  D2
```

라즈베리파이는 SPI0(GPIO 8/9/10/11) + INT(GPIO25)에 연결하고 `/boot/firmware/config.txt`에
`dtoverlay=mcp2515-can0,oscillator=8000000,interrupt=25` 추가 (모듈 크리스털이 8MHz인지 16MHz인지 확인 필수 — 틀리면 통신 안 됨).
