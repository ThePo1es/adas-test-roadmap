# Week 4 — 캡스톤: AEB 시험 시나리오 설계 + 시뮬레이션 검증

**주간 목표:** 시험 직무의 전체 사이클(요구사양 → Test Case → 시험 실행 → 자동판정 → 리포트)을 혼자 한 바퀴.
**이 폴더가 포트폴리오의 메인이다.**

시나리오: 자차(ego)가 v0로 주행, 전방 선행차 존재. TTC(Time-To-Collision)가 임계값 이하이면 AEB가 최대 감속으로 제동.
Week 3의 종방향 모델 + Euro NCAP CCR 시나리오(`ncap_scenarios.md`)를 재사용.

---

## Day 1 (월) — 요구사양 작성

| 시간 | 할 일 |
|------|------|
| 0:00~1:00 | 좋은 요구사항의 조건 학습: 측정 가능(수치), 검증 가능(시험으로 확인), 모호어 금지("잘", "적절히" 금지) |
| 1:00~4:00 | `requirements.md` 작성 — AEB 기능 요구사항 5개. 예시: "REQ-01: CCRs, 자차 30km/h, 선행차 정지 조건에서 충돌 없이 정지해야 하며 최대감속도는 9.8m/s² 이하" 형식으로 ID 부여. 작동 조건(속도 범위), 미작동 조건(오작동 방지), 감속도 한계, 경고 타이밍, 최소 정지거리 포함 |
| 4:00~4:30 | 각 요구사항이 "시험으로 검증 가능한가" 셀프 체크 — 안 되면 다시 쓰기 |
| 4:30~5:00 | 커밋 |

**완료 기준:** 5개 REQ 전부에 수치가 들어 있다.

## Day 2 (화) — Test Case 설계

| 시간 | 할 일 |
|------|------|
| 0:00~1:00 | TC 양식 정의: TC-ID / 연계 REQ-ID / 사전조건 / 입력(시나리오, 속도) / 절차 / 합격기준(충돌여부, 최종 gap, 최대감속도) |
| 1:00~4:00 | `test_cases.md` 작성 — CCRs/CCRm/CCRb × 자차속도 3단계(예: 20/40/60 km/h) = **9개 TC**. NCAP 프로토콜의 실제 속도 매트릭스 참고. 각 TC가 어느 REQ를 검증하는지 추적성(traceability) 표 포함 |
| 4:00~4:30 | 경계값 TC 1개 추가: "AEB 작동 최저속도 바로 아래"에서는 작동 안 해야 함 (미작동 검증도 시험이다) |
| 4:30~5:00 | 커밋 |

**완료 기준:** 10개 TC, 전부 REQ와 연결, 합격기준이 수치다.

## Day 3 (수) — 시뮬레이터 구현 1/2

| 시간 | 할 일 |
|------|------|
| 0:00~1:00 | `src/aeb_sim.py` 설계: Vehicle 클래스(위치 s, 속도 v), ttc 함수, AEBController 클래스, simulate 함수 — 뼈대와 입출력부터 종이에 |
| 1:00~3:30 | Vehicle + ttc 구현. ttc = gap / 상대속도 (접근 중일 때만 유한값, 아니면 inf). 단위 테스트: 손계산 값과 비교 3케이스 |
| 3:30~4:30 | 선행차 거동 구현: 정지(CCRs) / 등속(CCRm) / 제동(CCRb) 3종 |
| 4:30~5:00 | 커밋 |

**완료 기준:** AEB 없이 돌리면 충돌하는 시뮬레이션이 돌아간다 (gap이 0을 지나며 충돌 판정).

## Day 4 (목) — 시뮬레이터 구현 2/2

| 시간 | 할 일 |
|------|------|
| 0:00~2:00 | AEBController 구현: ttc < 임계값 → 감속 명령(-9.8·μ, μ=0.9). 히스테리시스(한 번 작동하면 정지까지 유지) 처리 |
| 2:00~3:00 | 시각화: 한 시나리오에 대해 거리/속도/TTC 3단 subplot, AEB 작동 시점 세로선 표시 |
| 3:00~4:00 | **검증:** 등속 접근 케이스에서 정지거리 시뮬레이션값 vs 해석해 v²/2a 비교, 오차 확인 |
| 4:00~4:30 | 파라미터 스터디: TTC 임계 0.8/1.2/1.6s 비교 → 이게 실무의 "성능육성"(빠르면 오작동↑, 늦으면 충돌↑ 트레이드오프) |
| 4:30~5:00 | 커밋 |

**완료 기준:** CCRs 40km/h에서 충돌 없이 정지하는 그래프가 나온다.

## Day 5 (금) — 시험 자동화 + 판정

| 시간 | 할 일 |
|------|------|
| 0:00~2:30 | `src/run_tests.py` 구현: 10개 TC를 리스트로 정의 → 전부 자동 실행 → 각 TC의 충돌여부/최종 gap/최대감속도 계산 → 합격기준과 비교해 PASS/FAIL |
| 2:30~3:30 | 결과물 생성: `results/summary.md`에 마크다운 표(TC-ID, 시나리오, 속도, 결과, 수치), TC별 그래프 png 저장 |
| 3:30~4:30 | FAIL이 있으면 원인 분석: 임계값 문제인지, 물리적으로 불가능한 조건인지(고속 CCRs는 물리 한계로 FAIL이 정상일 수 있음 — 그걸 리포트에 쓰는 게 진짜 시험 업무) |
| 4:30~5:00 | 커밋 |

**완료 기준:** 명령 한 번(`python run_tests.py`)으로 표와 그래프가 전부 생성된다.
